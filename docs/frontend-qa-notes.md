# 前端 QA 笔记

本文档记录当前前端页面的走查重点、客户演示风险点和建议修复优先级。当前页面以 hash 路由组织，主要入口包括首页、详情页、架构页和管理页；管理页内包含运营控制台、生产监控、支付运营、来源表单、Caps 输出、同步日志、低置信度审核和 TMDB 导入。

## 走查范围

- 首页：`#/`
- 详情页：`#/media/:id`
- 架构页：`#/architecture`
- 管理页：`#/admin`
- 来源表单：管理页内 `#sourceForm`
- Caps 输出：管理页内 `#capsOutput`
- 同步日志：管理页内同步日志面板
- 移动端断点：`max-width: 980px` 与 `max-width: 640px`

## 首页检查

当前首页包含顶部导航、左侧筛选、合规策略说明、Hero 区、统计卡片和资源目录卡片。客户演示时首页是第一印象，需要重点确认信息密度、筛选反馈和图片加载状态。

- 搜索框输入会立即触发重新渲染，并在渲染后尝试重新聚焦；需要检查快速输入时光标是否跳动、中文输入法组合输入是否被打断。
- 类型筛选和题材筛选依赖接口返回结果；需要确认无结果时页面不会只显示空白，应有明确空状态。
- Hero 区使用 `./public/pr.png`，需要确认部署路径下图片可访问；图片加载失败时目前没有备用说明或占位图。
- 统计卡片依赖 `/api/stats`，接口失败时当前路由没有页面级错误兜底，客户演示前应准备可用数据。
- 资源卡片标题、英文名、年份和 badge 都在卡片内展示；长标题、多语言标题和资源数较大时需要检查换行是否压迫海报。

## 详情页检查

详情页包含背景图、海报、返回链接、元数据、简介、资源筛选、同步按钮和资源表格。演示中通常会从首页点击进入详情，并触发一次同步来源。

- 背景图和海报均来自媒体数据字段；图片地址为空或失效时，首屏视觉会明显降级。
- 标题、原名、年份、时长、评分、题材和国家未做字段缺失兜底；需要检查接口返回不完整数据时是否出现 `undefined`。
- 简介较长时可能推高首屏高度，需要确认移动端不会遮挡资源区入口。
- “同步来源”按钮会先显示“同步中”，成功后显示“已导入 N”，随后重新渲染；如果同步失败，按钮文本会变成错误消息，长错误文本可能撑开工具栏。
- 资源表格列较多，目前依赖横向滚动；需要确认桌面端表头固定感、移动端横向滚动提示和长资源标题的可读性。
- 清晰度和字幕筛选会重新请求资源列表；需要检查筛选组合无结果时 `暂无匹配资源` 是否清晰。

## 管理页检查

管理页承担来源管理、状态检查、Caps、同步日志、低置信度审核和 TMDB 导入，信息量较大，客户演示前需要优先保证关键操作可解释、失败可恢复。

- 页面加载依赖 `/api/sources`、`/api/sync-logs`、`/api/review/resources` 三个接口；任一失败时当前没有局部降级显示。
- 来源列表每个来源显示名称、类型、地址、启用状态和操作按钮；地址较长时已设置 `overflow-wrap: anywhere`，仍需检查按钮区是否被挤压。
- 刷新按钮已改成显式重新请求管理页数据，演示时可直接点击刷新状态。
- 删除来源已经增加二次确认；演示时仍建议避免对真实来源执行破坏性操作。
- 测试来源失败时按钮文本直接显示错误消息，长错误会影响操作区布局。
- 低置信度审核的“通过/拒绝”没有确认和处理中状态；网络慢时用户可能重复点击。
- TMDB 未配置时会在结果区显示提示；已配置时需要确认搜索结果标题、年份和类型足够明确，避免导入错误条目。

## 来源表单检查

来源表单支持新增和编辑来源，字段包括来源 ID、名称、类型、权重、Base URL、RSS URL、API Key 和启用状态。

- 来源 ID 和名称为必填；Base URL、RSS URL、API Key 未按来源类型做条件校验，错误配置可能要到测试或同步时才暴露。
- 编辑来源时 API Key 使用后端脱敏值；保存时脱敏值会保留原凭据，客户演示和录屏仍需避免展示真实来源名称和 URL。
- 保存后会 `form.reset()` 并重新渲染；没有保存成功提示，用户可能不知道配置是否已落盘。
- 权重字段是数字输入，允许 `step=0.1`，但缺少最小值、最大值和非法值提示。
- 类型选项包含 `api`、`private`，但当前运行手册重点是 `internal`、`torznab`、`rss`；演示时需要避免选择尚未完整说明的类型。
- 启用来源默认未勾选；新增来源后如果忘记启用，同步结果可能看起来“没有生效”。

## Caps 检查

Caps 输出用于验证 Torznab / Prowlarr / Jackett 等来源能力，当前以 JSON 文本输出到深色 `pre` 面板。

- 点击 Caps 后显示“读取中...”，成功后输出格式化 JSON；需要确认大型 caps 响应不会让页面滚动体验过差。
- 失败时直接显示错误消息；建议错误中避免包含 API Key、完整授权 URL、Cookie 或 passkey。
- 当前 Caps 面板没有标识“正在查看哪个来源”，连续点击多个来源时客户可能不知道结果归属。
- Caps 输出没有复制按钮、清空按钮和折叠能力；演示时长 JSON 可能分散注意力。
- 如果来源类型不支持 Caps，应返回可理解提示，而不是空对象或通用请求失败。

## 同步日志检查

同步日志在管理页展示最近 8 条记录，详情页的“同步来源”按钮会触发新的同步记录。

- 日志项展示状态、类型、创建时间、导入数量和错误数量；缺少媒体标题、来源名称和可展开错误摘要。
- 时间字段直接展示接口返回的 `createdAt`，需要确认时区和格式对客户可理解。
- 当前只展示最近 8 条，没有分页、刷新时间和空状态以外的辅助信息。
- 如果同步部分成功、部分失败，日志应能表达“成功导入多少、哪些来源失败”，避免客户误认为整体失败。
- 错误数量为 0 不代表没有过滤或低置信度待审核；演示时可结合低置信度审核面板说明。
- 日志中不得出现 API Key、Cookie、passkey、下载 URL 或其他敏感信息。

## 移动端检查

当前 CSS 在 980px 下将主布局、Hero 和详情布局改为单列，在 640px 下进一步压缩顶部栏、表单和管理面板。

- 顶部导航在 640px 下纵向排列，需要检查品牌区和导航链接是否占用过多首屏高度。
- 首页侧边栏在 980px 下设置 `order: 2`，移动端会出现在内容之后；需要确认筛选入口是否仍然容易找到。
- Hero 图文在移动端改为单列，图片可能占据较大空间；演示手机视口时应确认资源目录不被过度下推。
- 统计卡片、资源卡片、架构层级和管理面板在小屏均改为单列；长内容不会横向撑破页面，但页面长度会明显增加。
- 详情页资源表格仍然是表格横向滚动；需要检查用户是否能发现可横滑，尤其是“活跃”等右侧列。
- 管理页来源操作按钮会换行，长错误文本仍可能导致按钮区拥挤。
- TMDB 结果和低置信度审核卡片在小屏下需要检查按钮是否有足够触控面积。

## 客户演示高风险 UI 点

- 敏感信息暴露：API Key 已脱敏，但 Caps 或错误消息仍可能带出授权 URL、Cookie 或 passkey。
- 破坏性操作：删除来源已有确认，但生产演示仍应使用演示数据。
- 接口失败缺少页面级兜底：首页、详情页和管理页的异步加载失败时可能停留在加载文案或只显示错误到按钮。
- 长错误消息破坏布局：测试来源、同步来源、Caps 失败时错误直接进入按钮或 `pre` 文本。
- 快速搜索体验不稳定：首页关键词输入每次变更都会重渲染，中文输入法和快速输入可能出现焦点或输入中断。
- 运营控制台数据需要平台管理员权限；非平台管理员可能看不到生产监控和全局状态。
- 空状态信息不足：无搜索结果、无资源、无日志、无审核项都有基础提示，但缺少下一步引导。
- 移动端表格可发现性弱：资源表格横向滚动没有明显提示，客户可能看不到完整列。
- 图片资源依赖路径：Hero 图、海报、背景图若加载失败，会直接影响首屏可信度。
- 操作反馈偏短：保存来源、导入 TMDB、审核通过/拒绝缺少 toast 或稳定成功状态。

## 建议修复优先级

### P0：演示前必须处理

- 为路由级接口请求增加错误兜底，至少显示“重试”入口和可理解错误摘要。
- 限制按钮错误文本长度，错误详情放到邻近提示区或面板内，避免撑破布局。
- 确认演示环境图片资源可访问，并准备图片加载失败占位样式。

### P1：建议近期修复

- 首页搜索增加 debounce，并兼容中文输入法 composition 事件。
- 管理页刷新按钮改为显式重新请求数据，而不是依赖 hash 链接。
- 来源表单按类型做条件校验：`torznab` 要求 Base URL，`rss` 要求 RSS URL，需要凭据的来源给出明确提示。
- Caps 面板显示当前来源名称、读取时间，并提供复制和清空按钮。
- 同步日志增加来源名称、媒体标题、部分失败摘要和可展开详情。
- 详情页同步按钮失败后恢复默认文案，并在旁边显示错误提示。
- 移动端资源表格增加横向滚动提示或改为卡片式资源列表。

### P2：体验增强

- 为保存来源、测试来源、TMDB 导入、审核通过/拒绝增加统一 toast 或状态栏反馈。
- 首页和详情页增加图片懒加载、错误占位和骨架屏。
- 管理页将来源列表、Caps、同步日志、审核和 TMDB 导入拆成 tabs 或更清晰的分区锚点。
- 资源表格增加排序说明、评分解释和来源可信度提示。
- 同步日志增加筛选、分页、自动刷新和失败日志导出。
- 架构页补充当前已实现能力与规划能力的视觉区分，避免客户误解原型范围。

## 演示前快速清单

- [ ] 首页统计、筛选和资源卡片能稳定加载。
- [ ] 至少一个媒体详情页有可展示的海报、背景图、简介和资源列表。
- [ ] 至少一个来源可测试成功。
- [ ] Caps 输出不包含敏感凭据。
- [ ] 手动同步后同步日志能出现新记录。
- [ ] 低置信度审核有可演示数据，或空状态文案清楚。
- [ ] TMDB 未配置或已配置两种状态的演示口径已准备。
- [ ] 375px、640px、980px、桌面宽屏至少各检查一次布局。
- [ ] 删除来源等破坏性操作在演示时避免误触，或已加确认。

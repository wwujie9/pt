# 客户可试用交付清单

本文档用于客户试用交付前的内部核对和客户现场演示准备。所有条目应以“客户可以按脚本完成试用”为目标，不以研发环境可运行作为交付完成标准。

## 1. 功能验收

### 1.1 基础流程

- [ ] 客户可访问试用入口，并能看到清晰的登录、首页或工作台。
- [ ] 核心业务流程可从头到尾完成，至少覆盖创建、查看、更新、删除或对应业务等价操作。
- [ ] 关键页面在正常数据、空数据、异常数据三类状态下均有可理解的展示。
- [ ] 表单输入有必要校验，错误提示能指向具体字段或具体操作。
- [ ] 列表、详情、筛选、搜索、分页等常用交互不会阻塞客户完成验收。
- [ ] 重要操作有确认、成功反馈或失败反馈，避免客户无法判断操作结果。

### 1.2 数据与状态

- [ ] 试用环境已准备一组演示数据，数据内容不包含真实敏感信息。
- [ ] 演示数据覆盖主要业务场景、边界场景和至少一个异常场景。
- [ ] 客户执行新增或修改操作后，刷新页面仍能看到一致结果。
- [ ] 客户误操作后的恢复路径明确，例如重新提交、撤销、重新导入或联系支持。
- [ ] 时间、金额、数量、状态等关键字段格式符合客户所在地区或合同约定。

### 1.3 兼容性

- [ ] 已确认客户计划使用的浏览器版本或运行环境。
- [ ] 在目标分辨率下无主要文字遮挡、按钮不可见或布局错位。
- [ ] 移动端或窄屏访问如在试用范围内，已完成关键路径检查。
- [ ] 如依赖第三方服务，第三方服务的试用账号、额度和网络访问均已确认。

## 2. 部署步骤

### 2.1 部署前准备

- [ ] 确认交付版本号、构建产物、配置文件和部署目标环境。
- [ ] 确认试用环境域名、端口、证书和网络访问策略。
- [ ] 确认运行时依赖已安装，包括数据库、缓存、对象存储、消息队列或外部 API。
- [ ] 确认环境变量已按试用环境填写，密钥不提交到代码仓库或文档。
- [ ] 确认数据库迁移脚本、初始化脚本和回滚脚本位置。

### 2.2 建议部署流程

1. 拉取或接收本次交付版本产物。
2. 备份当前试用环境配置和数据。
3. 安装依赖并执行构建流程。
4. 执行数据库迁移或初始化数据导入。
5. 写入试用环境配置和密钥。
6. 启动服务并检查进程、端口、日志和健康检查接口。
7. 使用试用账号完成一次核心业务流程冒烟验收。
8. 记录部署时间、版本号、操作者和关键配置变更。

### 2.3 部署后检查

- [ ] 健康检查接口返回正常状态。
- [ ] 前端资源、接口请求和静态资源均能正常访问。
- [ ] 服务日志无持续性错误、异常重启或权限失败。
- [ ] 数据库连接、外部来源连接和文件上传下载能力正常。
- [ ] 试用账号权限符合预期，不能访问超出试用范围的数据或功能。

## 3. 来源接入验收

### 3.1 来源配置

- [ ] 已明确客户试用阶段要接入的数据来源清单。
- [ ] 每个来源都有负责人、访问方式、认证方式、数据范围和刷新频率。
- [ ] 来源凭证通过安全渠道交付，并已配置到试用环境密钥管理位置。
- [ ] 来源字段映射已确认，必填字段、枚举值和数据类型均有对应关系。
- [ ] 来源异常处理策略明确，包括超时、限流、鉴权失败、数据为空和格式不一致。

### 3.2 接入验证

- [ ] 可成功连接每个来源，并能读取最小验收数据集。
- [ ] 来源同步或导入过程有日志，能定位失败来源和失败原因。
- [ ] 重复导入不会产生不可接受的重复数据或状态污染。
- [ ] 增量更新、全量更新或手动刷新策略已被客户理解并接受。
- [ ] 来源数据进入系统后，关键字段在客户界面可追踪、可核对。
- [ ] 来源不可用时，客户界面有可理解的提示，不暴露内部错误堆栈。

## 4. 权限与安全最低要求

### 4.1 账号与权限

- [ ] 已创建客户试用账号，并按角色分配最小必要权限。
- [ ] 管理员账号、演示账号、只读账号如有需要已分离。
- [ ] 客户账号不能访问其他客户、生产数据或内部调试接口。
- [ ] 默认密码已强制修改，或通过一次性邀请链接完成首次登录。
- [ ] 账号停用、密码重置和权限回收流程明确。

### 4.2 数据安全

- [ ] 试用数据不包含未经授权的真实个人信息、商业秘密或生产密钥。
- [ ] 如必须使用真实数据，已完成脱敏、授权和保留期限确认。
- [ ] 传输链路使用 HTTPS 或客户认可的安全通道。
- [ ] 密钥、Token、数据库密码不出现在前端代码、日志、截图或交付文档中。
- [ ] 日志记录避免输出完整凭证、身份证号、手机号、邮箱等敏感字段。

### 4.3 运行安全

- [ ] 试用环境与生产环境隔离。
- [ ] 管理入口限制访问来源或启用额外认证。
- [ ] 上传文件类型、大小和存储位置有基本限制。
- [ ] 外部回调、Webhook 或开放 API 有签名、Token 或来源校验。
- [ ] 已明确试用结束后的数据清理、账号回收和密钥轮换安排。

## 5. 已知限制

交付前应将以下限制同步给客户，避免试用期望偏差：

- 当前试用版本只覆盖约定范围内的核心流程，不代表完整生产能力。
- 试用环境性能容量有限，不应用于压测、生产流量或大规模批量任务。
- 演示数据和客户实际数据可能存在字段差异，正式上线前需补充字段映射验收。
- 第三方来源的稳定性、额度、响应时间和权限变更可能影响试用体验。
- 部分异常场景仅提供基础提示，完整告警、审计和自动恢复能力需在上线前补齐。
- 如客户网络、代理、防火墙或单点登录策略未提前确认，可能影响现场演示。
- 试用阶段的审计日志、操作留痕和数据留存策略以当前配置为准，上线前需复核。

## 6. 客户演示脚本

### 6.1 演示前准备

- [ ] 确认演示环境 URL、账号、密码或邀请链接可用。
- [ ] 准备一组客户能理解的业务样例数据。
- [ ] 提前打开备用浏览器窗口、日志观察窗口和来源系统页面。
- [ ] 确认客户参会人员角色，按业务、技术、安全和决策视角调整讲解重点。
- [ ] 准备失败兜底方案，例如截图、录屏、备用数据或本地演示环境。

### 6.2 建议演示顺序

1. 说明本次试用目标和范围，只聚焦客户可验收的核心场景。
2. 使用客户试用账号登录，展示默认首页、导航和角色权限边界。
3. 展示核心对象或核心任务列表，说明数据来源和当前状态。
4. 新建一条业务记录，演示字段校验、保存反馈和详情查看。
5. 修改关键字段，刷新页面并确认状态保持一致。
6. 触发一次来源接入或数据刷新，展示同步结果和日志线索。
7. 演示异常或空状态，例如无数据、来源失败或权限不足。
8. 使用只读或低权限账号验证权限边界。
9. 回顾客户需要完成的验收动作、反馈渠道和试用截止时间。

### 6.3 客户试用验收问题

- [ ] 客户是否能独立完成核心业务流程？
- [ ] 客户是否认可数据来源、字段含义和刷新方式？
- [ ] 客户是否发现权限、流程或展示内容与实际业务不一致？
- [ ] 客户是否明确哪些能力属于试用范围，哪些能力需要上线前补齐？
- [ ] 客户是否确认下一步反馈联系人、反馈格式和决策时间点？

## 7. 上线前必须补齐项

### 7.1 产品与功能

- [ ] 完成全部正式上线范围的功能验收，不只覆盖试用脚本。
- [ ] 补齐错误处理、空状态、加载状态、批量操作和边界场景。
- [ ] 完成客户真实数据字段映射、枚举映射和历史数据迁移方案。
- [ ] 明确正式环境的容量、性能指标和可用性目标。
- [ ] 完成用户培训材料、运维手册和客户支持流程。

### 7.2 工程与运维

- [ ] 建立正式环境部署流水线和版本回滚流程。
- [ ] 完成数据库备份、恢复演练和迁移回滚验证。
- [ ] 接入监控、告警、日志检索和关键业务指标看板。
- [ ] 完成正式域名、证书、网络策略、防火墙和白名单配置。
- [ ] 明确故障响应分级、联系人、响应时间和升级路径。

### 7.3 安全与合规

- [ ] 完成正式权限模型复核和最小权限配置。
- [ ] 完成敏感数据识别、脱敏、加密、留存和删除策略。
- [ ] 完成安全扫描、依赖漏洞检查和高风险项修复。
- [ ] 完成审计日志、操作留痕和管理员行为追踪。
- [ ] 完成客户合同、数据处理协议、合规要求和上线审批。
